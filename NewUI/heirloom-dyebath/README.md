# Heirloom — The Mercer Legacy (Dye Bath)

A multi-generational family archive, read as living water. Every voice is an
heirloom — a drop of the author's dye in one vessel. Time is depth; the colour
the family makes together is the sum of everyone added. Built as a web app and
an installable **PWA**.

The background is a real-time **WebGL** render of dye diffusing in lit water; the
interface floats over it as glass and type. Six screens:

1. **The legacy** (home) — the live water; heirlooms pinned at their depth.
2. **A memory** — one entry, over frosted water.
3. **The colour of us** — an orb that is a *window into the live water*, plus each voice's share.
4. **Leave an heirloom** — compose, in your colour.
5. **Sealed** — a wax-sealed frosted card; the writing stays under the wax until its date.
6. **The volume** — a book whose cover is a snapshot of the water *at the moment of printing*.

## Run it

Requires Node 18+.

```bash
npm install
npm run dev        # http://localhost:5173
```

Build / preview the production PWA:

```bash
npm run build
npm run preview
```

To test "install to home screen" / offline, use `npm run preview` (or deploy) and
open it over `https` or `localhost` — the service worker and manifest are wired
through `vite-plugin-pwa`.

## Structure

```
src/
  main.tsx              app entry
  App.tsx               scene state + composition
  types.ts              SceneId
  styles.css            full-viewport shell + every screen
  water/
    fragmentShader.ts   the GLSL dye-in-water shader
    WaterCanvas.tsx     persistent WebGL background (pauses when tab hidden)
    capture.ts          snapshot the live water (used by the Volume cover)
  components/
    WaxSeal.tsx         the matte oxblood infinity wax
    SceneNav.tsx        bottom navigation
  scenes/               WaterScene, ReadScene, ColourScene, AddScene, SealedScene, VolumeScene
  data/family.ts        dyes, entries, contributions  ← swap for your backend
public/
  manifest is generated by vite-plugin-pwa
  favicon.svg, icons/*
```

All content lives in `src/data/family.ts`. The screens are declarative over it,
so wiring a real backend means replacing that module (and the hardcoded compose
sample) with live data.

## Production notes (not yet done)

- The shader is GPU-hungry. It already pauses when the tab is hidden. For
  low-end devices, add a reduced-resolution path, honour `prefers-reduced-motion`
  (CSS already disables UI motion), and ship a pre-rendered fallback loop.
- `preserveDrawingBuffer: true` is on so the Volume cover can snapshot the canvas;
  it carries a small cost. In production, snapshot once at preview/print time
  rather than keeping it on permanently.
- The colour engine (accumulated tone, each voice's share, depth/diffusion, and
  the release-on-date dissolve) is represented in the data layer but not yet
  computed from real entries.
- Fonts load from Google Fonts via `index.html`; self-host **EB Garamond** for
  full offline support.
