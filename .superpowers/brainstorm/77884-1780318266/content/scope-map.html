⚠ This is a large scope

60+ artboards. To make this manageable, we should treat each section as its own sub-project with its own spec and plan. The first question below is: what does "built exactly" mean to you?

Q1 — What does "built exactly" mean for this project?

This shapes everything — the approach, the timeline, the level of effort per section.

A

Pixel-faithful port

The live app should look exactly like the design canvas artboards — every spacing value, font size, and component behavior reproduced precisely from the JSX mockups.

B

Design-system faithful + functionally complete

Apply the correct tokens (bone/ink/warm, Source Serif 4, 0px radius, hairlines) and overall layout structure, but engineering the real UX where mockups use static placeholders.

C

Priority sections only, then iterate

Pick the 2-3 highest-impact sections (e.g., Today home + Adoption landing + PWA roles), get them exact, then expand to the rest in future passes.
