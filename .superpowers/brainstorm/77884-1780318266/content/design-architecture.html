Architecture design — Heirloom frontend build

Approach B · Foundation wave → three delivery waves · all 8 sections · design-system faithful + functionally complete

Cross-cutting constraints (apply to every screen)
● bone/ink/warm tokens only — no arbitrary colors
● Source Serif 4 + Inter + JetBrains Mono — no 4th face
● 0px radius default, 2px inputs, never >4px
● 1px hairlines only — no 2px borders
● No icon libraries — ∞ is the only glyph
● TapestryCanvas animate=true wherever cloth appears
● Motion: cubic-bezier(0.16,1,0.3,1) · 180/360/720ms only
● No toasts — inline status only
● No spinners — hairline progress bar

JSX reference files → live TypeScript

Each JSX file in loom3 2/ is the visual reference. We translate layout + tokens into the equivalent TSX page, wiring real data (useAuthStore, API calls) where the mockup uses static props. Static data stays as stubs when the API doesn't exist yet.

Testing gate per wave

npm run build (tsc + vite) must stay green after each wave. No new icon-library imports. Animated cloth visible in dev server before marking any tapestry screen done. Visual smoke of live site after each wave deploy.
