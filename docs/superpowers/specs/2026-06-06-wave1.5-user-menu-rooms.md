## 7. Responsive audit ‚Äî LetterRoom + VoiceRoom

The Wave 1 pages have no `@media` queries and have not been tested at 375px (iPhone SE) or 390px (iPhone 14).

### What to check and fix in both pages

1. **Horizontal overflow** ‚Äî no element should overflow the viewport at 375px. Use `overflow-x: hidden` on the root container only if content wraps naturally.
2. **Touch targets** ‚Äî all interactive elements (read/edit/play buttons) must have a minimum hit area of 44√ó44px. Use `padding` to expand tap targets without changing visual size.
3. **Font sizes** ‚Äî inputs and interactive text must be `font-size: 16px` minimum to prevent iOS auto-zoom on focus. LetterRoom has no inputs but VoiceRoom's audio element should be checked.
4. **Safe-area bottom** ‚Äî content `paddingBottom` must account for the BottomNav height (64px) plus `env(safe-area-inset-bottom, 0px)`. Both pages use `paddingBottom: 120` which covers this ‚Äî verify it is sufficient on iPhone 14 Pro Max (home indicator = 34px safe area).
5. **CTA and list rows** ‚Äî verify they don't clip or wrap awkwardly at 375px. The `maxWidth: 680` container is fine; verify `padding: clamp(24px, 5vw, 48px)` computes correctly at narrow widths.

## 8. Network-aware rendering

### Hook: `useReducedMotion()` (new, in `cloudflare/frontend/src/hooks/useReducedMotion.ts`)

Returns `true` when the user has requested reduced motion OR is on a slow/metered connection:

```typescript
export function useReducedMotion(): boolean {
  const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  const conn = (navigator as any).connection;
  const slowNetwork = conn
    ? conn.saveData || conn.effectiveType === '2g' || conn.effectiveType === 'slow-2g'
    : false;
  return prefersReduced || slowNetwork;
}
```

### Apply in LetterRoom and VoiceRoom

Replace hard-coded `animation: \`hl-fade 360ms ${EASE}\`` with a conditional:

```typescript
const reducedMotion = useReducedMotion();
// ...
animation: reducedMotion ? undefined : `hl-fade 360ms ${EASE}`,
```

This affects:
- LetterRoom: the expanded letter body reveal animation
- VoiceRoom: the expanded audio player reveal animation

### VoiceRoom audio lazy-load on slow connections

On `saveData` or `2g`, do not set `autoPlay` on the audio element and display a plain "tap to load" affordance before mounting the `<audio>` element:

```typescript
const { reducedMotion, slowNetwork } = useReducedMotion(); // extend hook to return both
// In render:
{isPlaying && entry.fileUrl && (
  slowNetwork
    ? <audio src={entry.fileUrl} controls ref={...} onEnded={...} />  // no autoPlay
    : <audio src={entry.fileUrl} controls autoPlay ref={...} onEnded={...} />
)}
```
