## 7. Mobile layout вЂ” LetterRoom + VoiceRoom

All loom pages use ClothShell, which already handles `env(safe-area-inset-top)` (topbar height `calc(56px + env(safe-area-inset-top, 0px))` + scroll offset). Safe-area-inset-top is not a gap. The outstanding mobile work is basic layout correctness at narrow widths.

### What to audit and fix in both pages

1. **Touch targets** вЂ” all interactive elements (read/edit/play buttons) must have a minimum tap target of 44Г—44px. Use `padding` to expand the hit area without changing visual size.
2. **Horizontal overflow** вЂ” no element should overflow the viewport at 375px. Remove any fixed widths that would cause clipping; the `maxWidth: 680` container with `padding: clamp(24px, 5vw, 48px)` should handle this вЂ” verify it computes correctly.
3. **Safe-area bottom** вЂ” content `paddingBottom` must account for BottomNav (64px) plus `env(safe-area-inset-bottom, 0px)`. Both pages use `paddingBottom: 120` вЂ” verify it is sufficient on iPhone 14 Pro Max (home indicator = 34px).
